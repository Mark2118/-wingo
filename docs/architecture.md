## 2. 核心数据流

```
用户打开 landing.html
       │
       ▼
看到定价 → 点击"开始试用"
       │
       ▼
进入 register.html → 输入邮箱密码 → POST /api/auth/register
       │                                               │
       │                                               ▼
       │                                    创建用户 + 默认免费订阅
       │                                               │
       │                                               ▼
       │                                    返回 JWT Token
       │                                               │
       ▼                                               │
注册成功自动登录 ←─────────────────────────────────────┘
       │
       ▼
跳转 /index.html（主应用）
       │
       ▼
┌─────────────────────────────────────────┐
│ 1. 用户输入："做一个餐饮会员管理系统"     │
│                                         │
│ 2. POST /api/chat → 创建项目            │
│                                         │
│ 3. GET /api/stream → SSE 流式 PRD       │
│    PRD 实时渲染在对话气泡中               │
│                                         │
│ 4. PRD 生成完 → 自动启动 Pipeline       │
│    (asyncio.create_task)                │
│                                         │
│ 5. WebSocket /ws/{pid} → 推送进度       │
│    需求分析 → 架构设计 → 代码生成        │
│                                         │
│ 6. 代码生成阶段 → 右侧预览面板自动滑出   │
│    iframe 通过 /api/projects/{pid}/file  │
│    获取 index.html 内容实时渲染          │
│                                         │
│ 7. Pipeline 完成 → 部署成功             │
│    项目保存到 sidebar 列表               │
│                                         │
│ 8. 用户可继续对话修改 / 下载 / 预览      │
└─────────────────────────────────────────┘
       │
       ▼
免费版 3 个项目后 → 弹出升级提示
       │
       ▼
点击升级 → pricing.html → 选择套餐
       │
       ▼
POST /api/billing/subscribe → 创建订单
       │
       ▼
扫码支付 → 回调 /api/billing/webhook
       │
       ▼
更新 subscription 表 → 解锁权限
       │
       ▼
刷新页面 → 订阅状态生效 → 继续创建项目
```

                       ▼

         ▼
    ┌──────────────┐
    │ 功能权限检查  │ ←─ 免费版限制项目数 / 专业版解锁功能

## 5. 关键技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 前端框架 | 纯 HTML/CSS/JS | 单页应用足够，不需要引入 Vue/React 复杂度 |
| 数据库 | SQLite (P0) | 一人OPC，不需要维护 PostgreSQL |
| 认证 | JWT Bearer Token | 无状态，适合单页应用 |
| 密码加密 | bcrypt | 行业标准 |
| 实时通信 | SSE + WebSocket | SSE 用于单向流式，WebSocket 用于双向进度 |
| AI 路由 | 轻活Ollama / 重活Kimi | 本地优先，成本控制 |
| 部署 | 本地目录 + 可选SSH | 私有化部署为主 |
| 支付 | 模拟支付 (P0) | 先跑通流程，再接入微信/支付宝 |
