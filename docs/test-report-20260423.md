## 发现的 Bug（按严重程度）

### 🔴 严重

**1. `POST /api/projects` 缺少项目数量限制**
- 免费版3个项目限制只在 `/api/chat` 实现
- 直接调 `/api/projects` 可以绕过限制创建无限项目
- **修复**: 在 `projects.py` 的 `create_project_route` 添加限制检查

**2. 超长项目名称导致 500 Internal Server Error**
- 5000字符名称超出数据库字段长度 String(200)
- **修复**: API 层添加长度校验

### 🟡 中等

**3. XSS 项目名原样存储**
- `<script>alert(1)</script>` 作为项目名成功创建
- 依赖前端 escapeHtml，API 直出时有风险
- **修复**: 后端存储前做基本清洗

**4. `POST /api/projects` 使用 Query 参数而非 JSON Body**
- 不符合 REST 规范，中文需URL编码
- **修复**: 改为接收 JSON body

### 🟢 轻微

**5. 空 chat 文本也会创建项目**
- `text: ""` 时仍创建空项目
- **修复**: 添加空文本校验

**6. `projects.py` 缺少空名称校验**
- 可以创建 name="" 的项目
- **修复**: 添加非空校验
